## Complete before sending to a pilot firm

- [ ] Replace every `[COMPLETE BEFORE PILOT]` field.
- [ ] Deploy the verified build to the private staging URL.
- [ ] Confirm authentication, tenant isolation, storage, backup and deletion behaviour in staging.
- [ ] List the actual hosting, database, authentication, email, monitoring and AI sub-processors.
- [ ] Have the data-processing terms reviewed by a UK-qualified solicitor.
- [ ] Obtain signatures before receiving real client data.
- [ ] Name the ClosePilot pilot owner, technical support contact and incident contact.
- [ ] Create the pilot firm's users and test their access.
- [ ] Agree one client, one accounting period and the permitted file types.
- [ ] Ask the firm to remove data that is not needed for the review.
- [ ] Schedule the walkthrough and the seven-day follow-up.

## Pilot scope

The first pilot is deliberately narrow:

- one accounting firm;
- one nominated SME client;
- one accounting period;
- supervised use only;
- trial balance, P&L, balance sheet, aged debtors, aged creditors and VAT exports;
- review, evidence, manager approval, partner sign-off and review-pack export.

The pilot is successful when the partner understands the findings, can trace them to evidence, completes the review workflow and is willing to use the exported pack as part of a real client review.
